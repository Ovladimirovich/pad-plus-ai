# Как внести вклад в PAD+ AI

Спасибо за интерес к проекту! PAD+ AI — это открытая исследовательская платформа, и мы рады любой помощи: от тестирования до написания кода.

## 🚀 Быстрый старт

Не знаете, с чего начать? Вот три простых шага:

1. **Попробуйте приложение:** [https://padplus-ai.onrender.com](https://padplus-ai.onrender.com)
2. **Посмотрите задачи:** Issues с меткой [`help wanted`](https://github.com/Ovladimirovich/pad-plus-ai/issues?q=is%3Aissue+is%3Aopen+label%3A%22help+wanted%22)
3. **Для новичков:** Issues с меткой [`good first issue`](https://github.com/Ovladimirovich/pad-plus-ai/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22)

## 📖 Сначала ознакомьтесь с системой

В интерфейсе приложения есть **подробные инструкции** (кнопка "Инструкции" в меню). Там описано:

- 📋 Обзор системы
- 🚀 Быстрый старт
- 🖥️ Интерфейс и все страницы
- 💬 Чат и провайдеры
- 🧠 Память и RAG
- 😊 Эмоции PAD
- 🧬 HEALER (самодиагностика)
- 🔗 Граф знаний
- 🧪 Эксперименты

Прочитайте их перед тем, как тестировать или писать код — это сэкономит вам время.

## 🧪 Как помочь без программирования

### 1. Протестируйте приложение

Самый ценный вид помощи — найти баги и неудобства.

**Что проверять:**
- Сложные многошаговые запросы (работает ли X-Ray?)
- Длинные диалоги (помнит ли система контекст?)
- Попытки "сломать" Truth Loop (может ли система соврать?)
- Разные LLM-провайдеры (OpenRouter, GigaChat, OpenAI)
- Мобильная версия (всё ли удобно?)

**Как сообщить о проблеме:**
Используйте шаблон **Bug Report** при создании Issue. Он уже настроен и включает все нужные поля:
- Фазы пайплайна (safety → intent → retrieve → ...)
- Данные из X-Ray (Trace ID, Thought Stream)
- Информацию о HEALER
- Окружение (браузер, провайдер, модель)

### 2. Оставьте отзыв по интерфейсу

Не нужно искать баги — просто расскажите, что неудобно:
- Понятен ли интерфейс с первого взгляда?
- Полезна ли панель X-Ray или она перегружена?
- Легко ли найти нужные настройки?

Используйте шаблон **Пустой** или создайте Issue с меткой `ux`.

### 3. Переведите документацию

README на русском — это ограничение для международной аудитории. Нужны переводы на:
- English (приоритет)
- Spanish, Chinese, German, French

**Как перевести:**
1. Форкните репозиторий
2. Создайте `README.<lang>.md` (например, `README.en.md`)
3. Переведите, сохраняя markdown-форматирование
4. Добавьте ссылку на ваш перевод в основной README
5. Отправьте Pull Request

### 4. Улучшите документацию

Если нашли неточность в инструкциях или хотите добавить раздел — используйте шаблон **Документация проекта**.

## 🐛 Нашли баг?

Используйте шаблон **Bug Report** — он уже настроен под архитектуру PAD+ AI.

**Обязательно укажите:**
- ✅ Шаги для воспроизведения
- ✅ Фазу пайплайна, где произошел сбой
- ✅ Trace ID из X-Ray (если доступен)
- ✅ Провоайдер и модель
- ✅ Скриншоты
