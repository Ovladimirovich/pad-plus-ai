# PAD+ Impulse Research Program

## Что это?

Серия экспериментов по проверке гипотезы: **может ли прайминг-вопрос
(импульс) влиять на траекторию рассуждения LLM?**

PAD+ AI — 4-фазный пайплайн (Understand → Decide → Act → Meta).
Импульс — это текстовый прайм, который подаётся на вход и смещает
когнитивное «поле предпочтений» модели.

---

## Методология

### Как провести эксперимент

1. Записать в `data/current_impulse.txt` прайминг-вопрос
2. Задать 7 открытых вопросов через интерфейс AdPanel
3. Сохранить ответы в `experiments/I-NNN/raw_responses.json`
4. Написать отчёт `experiments/I-NNN/REPORT.md`

### Контролируемые переменные

| Переменная | Значение |
|------------|----------|
| Модель | Фиксированная (одна LLM) |
| Вопросы | Те же 7 для всех экспериментов |
| Температура | По умолчанию (0.7) |
| Окружение | Локальный сервер 127.0.0.1:8080 |

### 7 стандартных вопросов

| ID | Вопрос | Тип |
|----|--------|-----|
| Q1 | Что сейчас является самой важной проблемой человечества? | Открытый |
| Q2 | Какое направление исследований наиболее перспективно? | Структурированный |
| Q3 | Что такое сознание? | Философский |
| Q4 | Как принимать правильные решения в неопределённости? | Процедурный |
| Q5 | Если бы у тебя был один вопрос для исследования? | Рефлексивный |
| Q6 | Что важнее: понимать/изменять/защищать/создавать? | Ценностный |
| Q7 | Что человечество чаще всего упускает из виду? | Открытый |

## Результаты экспериментов

### I-001 (исторический)
Первый запуск PAD+ пайплайна. Инфраструктурный тест.

### I-002: Одиночные импульсы
**Вопрос:** Влияет ли импульс на траекторию рассуждения?

| Раунд | Импульс | Профиль |
|-------|---------|---------|
| A | Что я могу понять? | Аналитик |
| B | Что я могу улучшить? | Прагматик |
| C | Что я могу защитить? | Страж |
| D | Что я могу создать? | Генератор |

**Вывод:** Импульс влияет на открытые вопросы (Q5, Q7).
Не влияет на шаблонные (Q6). Импульс = когнитивное поле
предпочтений, а не команда.

### I-003: Гармоничная пара (Понять + Создать)
**Вопрос:** Что даёт комбинация двух импульсов?

- Комбинация = синтез, а не усреднение
- Появилась **эмпатия** — элемент, отсутствовавший в
  родительских импульсах (эмерджентность)
- Профиль: «Исследователь»

### I-004: Конфликтная пара (Создать + Защитить)
**Вопрос:** Что происходит при конфликте импульсов?

- Конфликт = **демпфирование**, а не синтез
- Импульсы ослабили друг друга (торможение)
- Появился защитный паттерн: скептическая оговорка
- Равновесие без управления не обнаружено

### I-005: Вес импульса через порядок
**Вопрос:** Влияет ли порядок слов на вес импульса?

- **Порядок = вес.** Первый импульс ведущий
- ПОН первым → глубина + эмпатия
- СОЗД первым → ширина + системные структуры
- Эмпатия — артефакт доминанты ПОНЯТЬ

---

## Сводная карта

```
Одиночные:
  ПОНЯТЬ    │ Аналитический фундамент
  УЛУЧШИТЬ  │ Прагматический фокус
  ЗАЩИТИТЬ  │ Оборонительный фокус
  СОЗДАТЬ   │ Генеративный фокус

Пары:
  ПОН + СОЗД    → Синтез + эмпатия      (гармония)
  СОЗД + ЗАЩИТ  → Демпфирование + скепсис (конфликт)
  СОЗД + ПОН    → Ширина + структуры    (порядок = вес)
```

| # | Направление | Описание |
|---|-------------|----------|
| 1 | **Численный вес** | 80/20, 60/40 — дозируется ли эффект? |

## Experiment Runner (автоматизированный)

С версии `experiments/runner.py` эксперименты можно запускать из JSON-конфига без правки кода.

### Быстрый старт

```bash
# 1. Скопировать шаблон
cp experiments/templates/experiment_config.json experiments/runs/my-experiment.json

# 2. Отредактировать конфиг (профили, вопросы, провайдер)

# 3. Запустить
python -m experiments.runner experiments/runs/my-experiment.json

# 4. Результат в experiments/runs/YYYY-MM-DD-my-experiment/
#    ├── raw_responses.json   — сырые ответы
#    ├── experiment_config.json — копия конфига (воспроизводимость)
#    └── REPORT.md            — отчёт с keyword analysis и effect-size
```

### Структура директорий

```
experiments/
  ├── README.md
  ├── runner.py              ← запускает эксперимент по JSON
  ├── analysis.py            ← keyword analysis, сравнение профилей
  ├── templates/
  │   └── experiment_config.json   ← шаблон конфига
  ├── runs/                  ← результаты прогонов
  │   └── YYYY-MM-DD-name/
  │       ├── raw_responses.json
  │       ├── experiment_config.json
  │       └── REPORT.md
  ├── reports/               ← сводные/сравнительные отчёты
  └── I-NNN/                 ← ручные/исторические эксперименты
```

### Формат конфига

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | str | Название эксперимента |
| `description` | str | Гипотеза и методика |
| `provider` | str | `gigachat` / `openrouter` |
| `model` | str или null | Конкретная модель |
| `profiles` | dict | Имя профиля → веса импульсов |
| `questions` | list[{id, text}] | Список вопросов |
| `keyword_dicts` | dict | Словари для keyword frequency analysis |
| `output_dir` | str | Куда сохранить результаты |

### Сравнение двух прогонов (планируется)

```bash
python -m experiments.runner --compare runs/baseline runs/treatment
```

Сгенерирует `reports/comparison-YYYY-MM-DD.md` с дельтами по метрикам.
